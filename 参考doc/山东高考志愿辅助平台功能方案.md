# 山东高考志愿辅助平台功能方案

## 一、产品核心流程

```text
填写分数与选科
→ 自动匹配山东一分一段位次
→ 筛选符合选科要求的招生专业
→ 展示院校及专业录取数据
→ 显示录取概率和冲稳保
→ 加入志愿表
→ 调整志愿顺序
→ 保存或导出志愿方案
```

系统推荐和加入志愿表的最小单位为：

**学校 + 招生专业（专业类）+ 招生类型 + 校区**

院校只用于对专业结果进行分组展示。

---

## 二、考生信息页

用户首次使用时填写：

| 字段 | 说明 |
|---|---|
| 高考年份 | 默认当前年份 |
| 高考分数 | 用户输入 |
| 高考位次 | 根据一分一段表自动计算，也允许手动修改 |
| 选考科目 | 从6门科目中选择3门 |
| 报考层次 | 本科、专科、不限 |
| 意向地区 | 可多选 |
| 意向专业 | 可选择专业门类、专业类或具体专业 |
| 排除专业 | 不希望报考的专业 |
| 学费上限 | 可不填写 |
| 院校性质 | 公办、民办、不限 |
| 中外合作 | 接受、不接受、不限 |

修改分数、位次或选科后，系统重新生成推荐结果和录取概率。

---

## 三、智能推荐页

### 3.1 页面顶部

固定展示当前考生信息：

```text
分数：600
位次：约23,000
选科：物理、化学、生物
候选专业：1,286个
```

支持快速修改分数、位次和选科。

### 3.2 筛选条件

支持以下筛选：

- 冲、稳、保；
- 录取概率范围；
- 院校名称；
- 专业名称；
- 院校所在地；
- 山东省内、山东省外；
- 本科、专科；
- 公办、民办；
- 双一流等院校标签；
- 专业门类和专业类；
- 学费范围；
- 中外合作；
- 校企合作；
- 当年招生人数；
- 招生人数增加、减少、新增；
- 历史最低位次范围。

### 3.3 排序方式

支持：

- 录取概率；
- 与考生位次的差值；
- 去年最低位次；
- 当年招生人数；
- 院校优先；
- 专业优先；
- 学费高低。

### 3.4 院校分组展示

搜索结果按院校分组。

院校卡片显示：

```text
山东大学
山东·济南｜公办｜本科｜双一流
符合条件专业：18个
概率范围：42%—96%
```

点击院校名称进入院校详情；点击展开按钮查看该校招生专业。

### 3.5 专业列表字段

每个专业显示：

| 字段 | 示例 |
|---|---|
| 专业名称 | 计算机科学与技术 |
| 招生类型 | 普通类 |
| 校区 | 青岛校区 |
| 选科要求 | 物理、化学 |
| 2026招生人数 | 12人 |
| 2025招生人数 | 10人 |
| 计划变化 | 增加2人 |
| 2025最低分 | 612分 |
| 2025最低位次 | 15,820 |
| 2024最低位次 | 16,430 |
| 2023最低位次 | 15,260 |
| 学制 | 4年 |
| 学费 | 6,600元/年 |
| 录取概率 | 73% |
| 风险标签 | 稳 |
| 数据状态 | 数据完整、新增专业、历史数据不足 |

每个专业提供以下操作：

- 查看详情；
- 查看专业介绍；
- 收藏；
- 加入志愿表。

## 四、专业详情页

专业详情页分为四个区域。

### 4.1 当前招生信息

展示：

- 学校名称；
- 专业或专业类名称；
- 招生代码；
- 招生类型；
- 所在校区；
- 当年招生人数；
- 选考科目要求；
- 学制；
- 学费；
- 招生备注；
- 单科、语种、体检等要求。

### 4.2 历年录取数据

展示近三年：

| 年份 | 招生人数 | 最低分 | 最低位次 |
|---|---:|---:|---:|
| 2025 | 10 | 612 | 15,820 |
| 2024 | 10 | 608 | 16,430 |
| 2023 | 8 | 615 | 15,260 |

同时显示位次趋势图和招生人数变化。

### 4.3 概率信息

展示：

```text
录取概率：73%
标签：稳
预测最低位次：约16,200
预测区间：15,300—17,100
```

概率下方展示算法返回的简短说明，例如：

```text
你的位次比预测最低位次靠前约1,400名；
今年招生计划增加2人；
近三年最低位次波动较小。
```

### 4.4 专业介绍外链

设置“了解这个专业”区域，提供：

1. **阳光高考专业介绍**
   - 跳转阳光高考专业知识库对应专业页面；
   - 无法匹配具体页面时，跳转阳光高考专业搜索页。

2. **该校专业介绍**
   - 优先跳转学校官网或学院官网的专业介绍页面；
   - 可包含培养目标、核心课程、培养方案等。

3. **学校招生官网**
   - 跳转该院校本科招生网；
   - 用于查看招生章程和最新通知。

页面按钮示例：

```text
[阳光高考专业介绍] [学校专业介绍] [学校招生官网]
```

数据表需要保存：

```text
sunshine_major_url
school_major_url
school_admission_url
school_official_url
```

其中阳光高考链接按标准专业维护，学校专业链接按“学校+专业”维护。

## 五、院校详情页

展示：

- 院校名称；
- 所在城市；
- 公办或民办；
- 院校类型；
- 院校标签；
- 学校官网；
- 招生官网；
- 阳光高考院校页面；
- 当年在山东招生的专业数量；
- 符合当前考生条件的专业列表。

院校详情中的专业列表沿用智能推荐页字段，并支持直接加入志愿表。

---

## 六、志愿表

### 6.1 基础功能

用户可以：

- 创建多份志愿表；
- 修改志愿表名称；
- 添加专业；
- 删除专业；
- 拖动调整顺序；
- 输入序号快速移动；
- 添加个人备注；
- 复制志愿表；
- 自动保存；
- 导出Excel。

### 6.2 志愿表字段

| 顺序 | 标签 | 概率 | 学校 | 专业 | 招生人数 | 去年位次 | 学费 | 备注 |
|---:|---|---:|---|---|---:|---:|---:|---|
| 1 | 冲 | 42% | 山东大学 | 临床医学 | 8 | 8,210 | 7,920 | 优先专业 |
| 2 | 稳 | 76% | 中国石油大学 | 计算机类 | 20 | 15,600 | 6,600 |  |
| 3 | 保 | 94% | 山东科技大学 | 软件工程 | 80 | 27,300 | 6,600 |  |

顶部显示：

```text
已添加：56 / 96
冲：18个
稳：24个
保：14个
```

### 6.3 添加逻辑

点击“加入志愿表”后：

1. 选择目标志愿表；
2. 默认加入志愿表末尾；
3. 已存在时提示“该专业已加入”；
4. 添加完成后允许继续浏览，不强制跳转；
5. 按钮状态变为“已加入”。

---

## 七、志愿表检查

点击“检查志愿表”，系统检查：

- 是否存在不符合选科要求的专业；
- 是否存在重复专业；
- 是否存在已经停止招生的计划；
- 是否全部为冲刺专业；
- 是否缺少保底专业；
- 是否包含用户排除的地区或专业；
- 是否超过学费上限；
- 是否包含中外合作或校企合作；
- 是否存在新增专业或历史数据不足；
- 是否存在特殊体检、语种或单科要求。

检查结果直接定位到对应志愿项。

## 八、数据管理后台

后台包含五个模块：

### 8.1 一分一段表

支持按年度上传Excel，字段包括：

```text
year
score
same_score_count
cumulative_count
```

### 8.2 当年招生计划

核心字段：

```text
year
school_code
school_name
major_code
major_name
major_category
campus
enrollment_type
education_level
subject_requirement
plan_count
tuition
duration
remark
```

### 8.3 历年录取数据

核心字段：

```text
year
school_code
major_code
school_name
major_name
enrollment_count
min_score
min_rank
```

### 8.4 院校和专业链接

维护：

```text
学校官网
学校招生官网
阳光高考院校链接
阳光高考专业链接
学校专业介绍链接
```

支持按专业代码批量匹配阳光高考链接，并允许人工修改。

### 8.5 数据校验

导入后检查：

- 学校代码是否存在；
- 专业代码是否存在；
- 招生计划是否重复；
- 选科要求是否为空；
- 招生人数是否异常；
- 历史分数和位次是否为空；
- 专业外链是否可访问；
- 同名专业是否属于不同招生类型或校区。

## 九、算法接口

主系统向算法服务传递：

```json
{
  "candidate": {
    "year": 2026,
    "score": 600,
    "rank": 23000,
    "subjects": ["物理", "化学", "生物"]
  },
  "plan": {
    "plan_id": 12345,
    "current_plan_count": 12,
    "history": [
      {"year": 2025, "plan_count": 10, "min_rank": 15820},
      {"year": 2024, "plan_count": 10, "min_rank": 16430},
      {"year": 2023, "plan_count": 8, "min_rank": 15260}
    ]
  }
}
```

算法服务返回：

```json
{
  "probability": 73,
  "label": "稳",
  "predicted_rank": 16200,
  "rank_range": [15300, 17100],
  "confidence": "高",
  "reasons": [
    "考生位次领先预测线约1400名",
    "招生计划增加2人",
    "历史位次波动较小"
  ],
  "model_version": "v1.0"
}
```

前端不负责计算冲稳保，只展示算法结果。

## 十、开发模块划分

### 产品

负责页面原型、字段定义、筛选逻辑、交互流程和验收标准。

### 数据

负责一分一段表、招生计划、历年录取数据、专业代码映射及外部链接维护。

### 开发

负责用户信息、推荐查询、专业详情、志愿表、后台导入及算法接口。

### 算法

负责预测概率、冲稳保标签、预测位次区间和原因说明。

---

## 十一、第一期交付范围

第一期完成：

1. 分数匹配位次；
2. 选科过滤；
3. 院校分组和专业展开；
4. 当年招生计划展示；
5. 近三年最低分和最低位次；
6. 专业介绍及学校官网外链；
7. 录取概率和冲稳保接口；
8. 专业加入志愿表；
9. 志愿表排序、删除和自动保存；
10. Excel数据导入后台；
11. 志愿表基础检查；
12. 志愿表Excel导出。

以上功能完成后，即形成可供家庭实际使用的完整闭环。
