# 山东 2026 志愿填报资料数据价值分析

> 分析时间：2026-07-05  
> 资料目录：`参考doc/14、山东-2026高考志愿填报资料/`  
> 目标：判断这些资料是否能支撑山东专科位次预测模型，哪些资料可直接入模，哪些只能做辅助规则或展示，以及仍缺哪些关键数据。

## 1. 总体结论

这批资料已经基本补齐了第一版建模系统最需要的三类核心数据：

1. **2026 招生计划**：有完整结构化表 `山东-2026-招生计划.xlsx`，并且专科三表合一已合并 2026 专科计划。
2. **2022-2025 历史专业级录取结果**：有 `2022-2025年全国高校在山东的专业录取分数.xlsx`，覆盖专业、最低分、最低位次、录取人数。
3. **2022-2025 历史招生计划**：有 `2022-2025年全国高校在山东的招生计划.xlsx`，覆盖专业、计划人数、学制、学费、选科要求。

因此，当前已经可以启动专科预测模型 MVP，不必等待掌上高考或 EOL 爬虫全部跑完。爬虫结果应转为“交叉校验数据源”，用于验证第三方表和官网计划的一致性。

但是，还不能说“所有数据缺陷都完全弥补”。最大缺口仍然是：

1. **本科第 1 次志愿录取后，各位次段还剩多少考生会进入专科竞争**：这类分段剩余考生流通常不会直接公开，只能用历史录取、剩余计划、分数段和投档线反推。
2. **2026 常规批第 2 次志愿实际剩余本科计划**：现有 2026 招生计划主要是初始计划，不等于本科第 1 次录取后正式剩余计划。
3. **第三方整理表的官方一致性校验**：部分文件很完整，但仍需要和山东考试院、阳光高考、学校官网章程做抽样校验。
4. **扫描型 PDF 的结构化规则**：色觉、高度近视、平行志愿规则、双高计划等 PDF 多为图片扫描，不能直接入库，需要 OCR 或人工结构化。

## 4. 对建模方案的影响

### 4.1 数据源优先级应调整

原计划里把 EOL/掌上高考爬取作为“完整计划”的主要来源。现在建议调整为：

1. **主数据源**：专科三表合一，先跑通 MVP。
2. **校验数据源**：`山东-2026-招生计划.xlsx`，校验计划数、学校代码、专业代码、批次和费用。
3. **交叉验证源**：EOL/掌上高考/夸克/学校官网，用于发现缺失、错行、变更和官方来源证据。
4. **权威修正源**：山东考试院后续公布的第 2 次志愿计划、剩余计划、投档结果。

每条训练样本建议增加：

| 字段 | 含义 |
| --- | --- |
| `source_name` | 数据来源文件或平台 |
| `source_priority` | 官方、半官方、第三方整理、推断 |
| `source_confidence` | 高/中/低 |
| `source_checked` | 是否和另一个来源交叉校验 |
| `raw_file_path` | 原始文件路径 |

### 4.2 第一版训练表可以更快搭建

专科三表合一已经包含以下关键字段：

- 2026 计划：年份、生源地、批次、科类、计划类别、院校代码、院校名称、专业代码、专业名称、专业备注、选科要求、计划人数、学制、学费、门类、专业类、是否新增。
- 历史录取：2025、2024、2023 的录取人数、最低分、最低位次、平均分、平均位次。
- 院校属性：所在省、城市、城市标签、院校标签、院校水平、隶属单位、类型、公私性质、本科/专科、保研率、院校排名。
- 专业属性：学科评估、专业水平、本专业硕士点、本专业博士点。

这意味着 MVP 可以先不做复杂多表 join，直接从三表合一生成第一版 `plan_series_features_2026.csv`。

但三表合一仍需注意：

1. 表头有分组行，历史字段有重复列名，导入时必须改成 `admit_count_2025`、`min_rank_2025`、`admit_count_2024` 等标准字段。
2. `是否新增` 列样例中混有“新增”和专业类值，需要检查是否存在错列或字段语义混杂。
3. 2025/2024/2023 字段之外，仍应和原始 `专业录取分数.xlsx` 交叉校验。
4. 计划类别里有高职专项、省属公费医学生等特殊项目，普通专科主模型需要分开处理。

### 4.3 新增建议影响因素

#### P0：第一版必须加入

| 因素 | 来源 | 原因 |
| --- | --- | --- |
| 历史最低位次、最低分、录取人数 | 专业录取分数、三表合一 | 最强信号 |
| 历史计划人数、今年计划人数、计划变化率 | 招生计划、三表合一 | 扩招缩招直接影响投档位次 |
| 计划类别 | 三表合一、2026 计划 | 3+2、中外合作、校企合作、专项计划行为差异很大 |
| 公办/民办、本科院校专科/高职院校 | 院校基础信息、三表合一 | 专科批强影响因素 |
| 学费、学制、选科要求 | 招生计划、三表合一 | 影响报考意愿和资格 |
| 城市、省份、院校类型 | 院校基础信息、三表合一 | 城市吸引力和地域偏好明显 |
| 新增专业、小计划标记 | 三表合一、历史计划 | 高波动风险 |

#### P1：第二版加入，提升解释和校准

| 因素 | 来源 | 建议用法 |
| --- | --- | --- |
| 双高计划标签 | `双高计划.pdf` OCR 后 | 高职院校吸引力加分 |
| 软科/GDI/院校排名 | 软科、GDI、全国高校名单 | 做学校层级弱特征，避免权重过高 |
| 专业满意度、就业满意度 | 专业满意度表 | 低权重特征，主要用于解释 |
| 专业就业行业、工资区间 | 专业就业信息 | 构建就业导向标签，不直接信任数值 |
| 招生章程限制 | 章程链接、PDF/OCR | 硬过滤和风险提示 |
| 体检色觉近视限制 | 限制专业 PDF | 硬过滤 |

#### P2：只做展示或人工参考

| 因素 | 原因 |
| --- | --- |
| 世界大学排名、QS、USNews | 对山东专科专业投档解释力弱 |
| 泛化热门专业 PDF | 不如山东历史录取数据可靠 |
| 五步法图片 | 可作为传统等位分解释，不应替代概率校准 |

## 5. 推荐的数据处理步骤

2. 2026 招生计划。

4. 2022-2025 招生计划。

```text

raw_score_rank_2022_2026.parquet
```

### Step C：统一口径

必须统一：

- `batch_std`：普通类一段、普通类二段、专科批、提前批、常规批第 2 次志愿。
- `education_level_std`：本科、职教本科、专科。
- `plan_type_std`：普通计划、3+2、中外合作、校企合作、高职专项、公费医学生、其他特殊。
- `school_code_std`：山东招生代码与部标代码分开保存，不能混用。
- `major_name_std`：去掉校区、方向、合作、备注后的标准专业名，同时保留原始全称。

### Step D：交叉校验 2026 计划

校验维度：

1. 按批次统计计划行数和计划数。
2. 按院校统计计划数。
3. 按 `院校代码 + 专业代码 + 专业全称` 比对三表合一和 2026 计划表。
4. 抽样和 EOL/掌上高考/学校官网章程核对。

不一致时处理：

- 官方文件优先。
- 第三方一致但官方缺失：标记 `source_confidence=medium`。
- 第三方冲突：进入人工复核表。

### Step E：构造专科历史面板

以 `school_code + major_code + major_name_std + plan_type_std` 为核心匹配键，生成：

```text
specialized_panel_2022_2026
```

字段包括：

```text
year, school_code, school_name, major_code, major_name_raw, major_name_std,
plan_type_std, plan_count, tuition, duration, subject_req,
admit_count, min_score, min_rank, avg_score, avg_rank,
school_attrs..., major_attrs..., source_confidence
```

### Step F：构建规则库

规则库分两层：

1. **硬过滤**：选科、性别、体检、色觉、单科、外语语种、特殊资格。
2. **风险提示**：高学费、小计划、新增、3+2 高分下沉、热门专业拥挤、历史波动大。

扫描型 PDF 需要先 OCR，不要直接假设已结构化。

### Step G：回测后再上线概率

已有 2022-2025 数据后，建议回测方式调整为：

1. 用 2022-2023 预测 2024。
2. 用 2022-2024 预测 2025。
3. 两轮回测通过后预测 2026。

如果要更稳，再补 2021 后做：

1. 用 2021-2023 预测 2024。
2. 用 2022-2024 预测 2025。

## 6. 是否还缺数据

还缺，而且这些缺口都和“2026 实际竞争环境”有关，不是普通资料包能完全解决的。

| 仍缺数据 | 优先级 | 为什么缺 | 解决方式 |
| --- | --- | --- | --- |
| 2026 本科第 1 次录取后各位次段剩余考生 | 最高 | 官方通常不会公布每个位次段剩余人数 | 用历史分段录取率、2026 本科计划、剩余本科计划、一分一段反推，发布区间而非单点 |
| 2026 常规批第 2 次志愿正式剩余本科计划 | 最高 | 需等待本科第 1 次录取后官方公布 | 建爬虫/手工导入山东考试院公告 |
| 2026 第 2 次志愿实际投档结果 | 高 | 录取后才会出现 | 用于赛后评估和 2027 模型 |
| 第 3 次志愿历史剩余计划与投档结果 | 中高 | 当前资料未见结构化表 | 继续找山东考试院历史公告，给断档/未满额标签用 |
| 2021 专业级计划和录取 | 中 | 当前核心历史从 2022 起 | MVP 不阻塞，做五年模型时再补 |
| 章程限制结构化数据 | 中 | 只有链接和扫描资料 | 爬章程 + OCR + 规则抽取 |

## 7. 对当前项目的执行建议

1. **立即把三表合一作为 MVP 主数据源**，先生成第一版专科预测特征表。
2. **保留 EOL/掌上高考爬虫**，但改成校验和补漏，不再阻塞模型启动。
3. **优先做数据质量报告**，特别是三表合一和 2026 计划表的一致性、重复行、计划数汇总、字段错列。
4. **先做 baseline 回测**，用 2022-2024 预测 2025，确认能优于“直接照搬去年最低位次”。
5. **不要过早承诺精确概率**，在本科第 1 次志愿结果出来前，只输出粗粒度概率和不确定区间。
6. **新增 `source_confidence` 和 `risk_tips`**，让系统能诚实解释哪些结论来自官方、哪些来自第三方整理、哪些来自模型推断。

## 8. 结论回答

截图里的这些文件夹，**对建模来说已经非常有价值，核心数据基本够启动第一版系统**。它们补齐了之前最麻烦的“2026 专科计划”和“2022-2025 专业级历史录取/计划”问题。

但它们不能完全替代 2026 录取过程中的实时数据。最后影响专科预测精度的关键，仍然是本科第 1 次录取后有多少高位次考生没有被本科吸收，以及这些人会不会下沉到 3+2、公办专科、医学、铁路、电力等热门专科专业。这个部分必须用 `rank_segment_flow` 模型估计，并在官方第 2 次志愿计划和录取结果公布后更新。
