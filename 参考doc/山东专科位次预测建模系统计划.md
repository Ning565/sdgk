# 山东专科位次预测建模系统计划

> 目标：面向山东夏季高考普通类常规批第 2 次志愿，建立“先计算去年专科等价总位次，再预测每个专业+学校今年最低投档位次和录取概率”的建模系统。

## 1. 思路判断

当前方案的方向是合理的，核心优点是把两个不同问题拆开处理：

1. **考生池变化问题**：本科第 1 次志愿录取后，哪些位次段的考生还会进入专科竞争。这部分用“有效专科竞争人数”和“去年专科等价总位次”解决。
2. **专业个体变化问题**：同样的总体竞争环境下，不同专业会因为公办/民办、3+2、城市、学费、专业热度、计划变化等因素受到不同冲击。这部分用专业级预测模型解决。

不把专科拆成“公办市场、民办市场、3+2 市场”等多个市场位次是对的。拆太细会造成样本稀疏，也会让用户难以理解。更稳妥的做法是保留一个总的专科等价位次，再把公办、3+2、城市、学费、专业热度等作为专业级特征进入模型。

最大风险不在 LightGBM、CatBoost 或其他算法选择，而在数据：如果无法可靠估计“本科第 1 次录取后，各位次段还剩多少人真正参与专科竞争”，等价位次只能是粗略估计。这个数据应作为项目最高优先级。

## 2. 建模范围

### 2.1 当前版本范围

- 省份：山东
- 类别：夏季高考普通类
- 志愿批次：常规批第 2 次志愿
- 投档单位：专业 + 学校
- 志愿数量：最多 96 个
- 投档规则：位次优先，遵循志愿
- 预测对象：专科计划，以及第 2 次志愿中仍可填报的剩余本科计划对专科竞争的影响

### 2.2 不直接覆盖的范围

- 常规批第 3 次志愿：可以复用同一框架，但必须重新计算考生池、剩余计划和专业级特征，不能直接沿用第 2 次志愿结果。
- 艺术类、体育类、春季高考：先不进入本模型。
- 提前批、高职专项、公费医学生等特殊项目：先作为资格过滤和特殊标签处理，不混入普通专科主模型。

## 3. 最终输出

### 3.1 单个考生输出

对每位考生先输出一个总体定位：

| 字段 | 含义 |
| --- | --- |
| `current_rank` | 2026 年考生真实总位次 |
| `effective_competition_position_2026` | 2026 年专科有效竞争池中的位置 |
| `equivalent_rank_2025` | 折算到 2025 年专科环境的等价总位次 |
| `equivalent_rank_low/high` | 等价位次误差区间 |
| `model_version` | 预估版或更新版模型版本 |
| `confidence` | 等价位次可信度 |

### 3.2 单个专业+学校输出

| 字段 | 含义 |
| --- | --- |
| `plan_id` | 今年招生计划 ID |
| `school_name` | 学校名称 |
| `major_name` | 专业名称 |
| `predicted_cutoff_rank_p10` | 偏难情形最低投档位次 |
| `predicted_cutoff_rank_p50` | 中性预测最低投档位次 |
| `predicted_cutoff_rank_p90` | 偏易情形最低投档位次 |
| `admission_probability` | 单项投档概率，按 5% 粒度展示 |
| `label` | 冲/稳/保/不建议 |
| `volatility_label` | 低波动/中波动/高波动 |
| `reason_codes` | 主要原因编码，如扩招、公办、3+2、学费高、计划小 |
| `risk_tips` | 风险提示，如新增专业、小计划、历史断档 |

### 3.3 96 志愿表输出

| 字段 | 含义 |
| --- | --- |
| `single_probability` | 不考虑志愿顺序时的单项投档概率 |
| `final_destination_probability` | 考虑 96 志愿顺序后的最终去向概率 |
| `slip_probability` | 全部滑档概率 |
| `inversion_warnings` | 志愿顺序倒挂提示 |
| `safety_coverage` | 保底是否充分 |
| `most_likely_destination` | 最可能录取的专业+学校 |

## 5. 当前缺少的数据

### 5.1 最高优先级缺口

1. **本科第 1 次录取后分位次段剩余考生估计**
   - 理想字段：每 1000 位或每个分数段的原始人数、已录取人数、剩余人数、预计参与专科人数。
   - 当前状态：不会被本次资料包直接补齐，需要通过历史录取、招生计划、一分一段和剩余计划反推。
   - 缺失影响：这是专科等价总位次可信度的最大瓶颈。

2. **2026 常规批第 2 次志愿正式剩余本科计划**
   - 需要字段：剩余本科专业+学校、计划数、选科要求、历史位次。
   - 当前状态：现有 2026 招生计划主要是初始计划，不等于本科第 1 次志愿录取后的正式剩余计划。
   - 缺失影响：无法估计本科剩余计划会吸走多少高位次考生。

3. **2026 计划官方一致性与版本冻结**
   - 当前状态：专科三表合一和 `山东-2026-招生计划.xlsx` 已基本补齐计划，但仍需和山东考试院、学校官网、EOL/掌上高考抽样比对。
   - 缺失影响：如果第三方整理表存在错行、漏行、计划数变化，会直接影响 `K_26` 和专业级预测。

4. **2026 实际投档结果**
   - 当前状态：录取后才会产生。
   - 缺失影响：不影响 2026 预测启动，但影响赛后评估和 2027 模型迭代。

### 5.2 中优先级缺口

1. **跨年专业系列匹配**
   - 需要把同一学校下跨年变化的专业名称、专业代码、校区、合作办学标签映射到同一个 `plan_series_id`。
   - 需要处理：改名、专业类拆分、专业合并、新增、停招。
   - 当前状态：专科三表合一已经做了部分合并，但必须和原始专业录取分、招生计划表复核，不能完全盲信。

2. **第 3 次志愿剩余计划和未满额标记**
   - 用于训练“是否断档、是否未招满、是否高波动”特征。
   - 当前资料未发现稳定结构化表，需要继续从山东考试院历史公告补。

3. **选科与体检/性别/外语限制结构化**
   - 用于硬过滤，不应只靠模型概率。
   - 当前状态：已有招生章程链接、色觉/高度近视 PDF、计划备注，但 PDF 多为扫描型，需要 OCR 或人工结构化。

4. **2021 历史专业+学校投档数据**
   - 当前已有 2022-2025，如果只做 MVP 和两轮回测已经够启动；如果要五年稳定模型，需要补 2021。
   - 缺失影响：小计划、断档、专业改名的长周期波动判断会偏弱。

### 5.3 低优先级但有价值的数据

- 学校就业质量、升本率、专业就业口径。本次资料已有一部分，就业类字段建议低权重使用。
- 学校所在城市吸引力指数。当前可先用城市标签、省份、是否省会等简化特征替代。
- 专业搜索热度或咨询热度。当前资料包未提供，后续可从平台搜索指数或用户行为补。
- 同校同类专业之间的相对排序。
- 历年学费变化。

这些数据可以提升解释和微调，但不应阻塞第一版模型。

## 7. 核心建模流程

## Step 0：统一口径

### 要做什么

明确所有数据只进入同一口径：

- 山东
- 夏季高考
- 普通类
- 常规批第 2 次志愿
- 专业 + 学校
- 最低投档位次

### 产出

- `batch_code` 映射表：第 1 次、第 2 次、第 3 次志愿的标准编码。
- `education_level` 标准值：本科、专科。
- `plan_type_tag` 标准值：普通、3+2、中外合作、校企合作、高职专项、公费医学生等。

### 验收标准

任意一条历史数据都能明确回答：

- 它属于哪一年？
- 属于第几次志愿？
- 是本科还是专科？
- 是不是普通类常规批？
- 最低位次是否可用？

## Step 1：历史投档线清洗

### 要做什么

从本地核心表构造历史样本：

- 优先读取专科三表合一，生成 2026 专科计划和 2025/2024/2023 历史特征。
- 读取 `2022-2025年全国高校在山东的专业录取分数.xlsx`，补齐 2022 历史最低位次和录取人数。
- 读取 `2022-2025年全国高校在山东的招生计划.xlsx`，补齐历史计划数、学费、学制、选科要求。
- 将 `all_toudangxian.csv` 作为已有仓库结构化结果，与新资料交叉校验。
- 保留 2022-2025 普通类二段/专科批/常规批第 2 次志愿专科数据。
- 标准化 `school_code`、`major_code`、`major_name`。
- 清理 `min_rank` 为整数。
- 标记异常：最低位次缺失、计划数缺失、计划数极小、明显断档。

### 产出

- `history_specialized_2022_2025_clean.csv`
- `raw_2026_specialized_3in1_clean.csv`
- 字段至少包括：`year, school_code, school_name, major_code, major_name, plan_count, min_rank, education_level, batch, plan_type, tuition, duration, subject_req, source_confidence`

### 验收标准

- 2022、2023、2024、2025 每年样本量合理。
- `min_rank` 缺失率可统计。
- 同一年同一 `school_code + major_code + major_name` 不重复。
- 三表合一、专业录取分数、历史招生计划之间的关键字段可追溯。

## Step 2：跨年 plan_series 匹配

### 要做什么

将历史和今年计划映射成同一招生单元：

优先匹配：

1. `school_code + major_code`
2. `school_code + major_name` 标准化后匹配
3. 同校相似专业名称匹配
4. 人工确认低置信度项

### 产出

- `plan_series_map.csv`
- 字段：`plan_series_id, school_code, standard_major_code, major_name_alias, match_confidence, match_status`

### 验收标准

- 最近三年连续招生的主流专业匹配率达到 95% 以上。
- 匹配置信度低于 0.85 的样本不直接进入模型训练，先进入人工复核队列。

## Step 3：构建有效专科竞争池

### 要做什么

按位次段估计真正进入专科竞争的人数。

位次段可以先用每 1000 名一段：

```text
segment = [rank_start, rank_end]
raw_count = rank_end - rank_start + 1
not_admitted_before_specialized = raw_count - admitted_before_specialized
remaining_undergraduate_absorbed = not_admitted_before_specialized * undergraduate_remaining_absorb_rate
specialized_participants = (not_admitted_before_specialized - remaining_undergraduate_absorbed) * specialized_participation_rate
```

有效竞争人数：

```text
effective_competitors(segment)
= raw_count
  * not_admitted_rate(segment)
  * (1 - remaining_undergraduate_absorb_rate(segment))
  * specialized_participation_rate(segment)
```

### 7 月 22 日前预估版

使用历史本科录取完成情况、今年本科计划、一段线人数、历史同位次段落榜率估计。

### 7 月 22 日后更新版

用实际本科第 1 次志愿录取结果、剩余本科计划、完整专科计划重新计算。

### 产出

- `rank_segment_flow_2026_pre.csv`
- `rank_segment_flow_2026_post.csv`
- 字段：`year, rank_start, rank_end, raw_count, admitted_est, remaining_undergraduate_absorb_est, specialized_participation_rate, effective_competitors`

### 验收标准

- 累计有效竞争人数曲线单调递增。
- 高位次段有效参与率低于低位次段，符合“高分考生更可能被本科吸收或只报优质专科”的常识。
- 与历史专科实际投档结果反推的参与规模不冲突。

## Step 4：计算专科等价总位次

### 要做什么

定义：

```text
D_y(r) = y 年在 r 位次之前的累计有效专科竞争人数
K_y = y 年有效专科计划数
pressure_y(r) = D_y(r) / K_y
```

对考生 2026 位次 `r_26`：

```text
pressure_26 = D_26(r_26) / K_26
equivalent_rank_2025 = argmin_r |D_25(r) / K_25 - pressure_26|
```

同时通过参数扰动生成区间：

- 乐观：本科吸收更多、专科参与率更低、专科计划更多。
- 中性：主模型估计。
- 悲观：本科吸收更少、专科参与率更高、专科计划更少。

### 产出

- 一个可调用函数：输入 `year, rank, subjects`，输出 `equivalent_rank_base/low/high`。
- 一个批量表：`equivalent_rank_lookup_2026_to_2025.csv`。

### 验收标准

- 位次越靠后，等价位次整体越靠后。
- 当今年专科扩招或本科吸收增加时，等价位次应向更容易方向移动。
- 等价位次区间宽度能反映数据不确定性，7 月 22 日后版本区间应明显收窄。

## Step 5：专业级特征工程

### 要做什么

为每个 `专业+学校` 构造特征。

### 历史特征

- 去年最低位次
- 前年最低位次
- 三年前最低位次
- 历史位次趋势
- 历史波动
- 是否曾断档
- 是否曾进入第 3 次志愿

### 计划特征

- 今年计划数
- 去年计划数
- 计划变化人数
- 计划变化比例
- 小计划标记：如计划数小于 5
- 新增/停招/恢复招生标记

### 院校专业属性

- 公办/民办
- 学校所在省、市
- 城市等级
- 学费
- 学制
- 是否本科院校专科
- 是否 3+2
- 是否中外合作/校企合作
- 专业类别
- 热门专业标签：口腔、护理、医学、电力、铁路、计算机、师范等
- 院校排名/院校层次：软科、GDI、双高计划、是否本科院校、是否公办
- 专业口碑和就业弱特征：专业满意度、就业满意度、就业行业、薪酬区间
- 硬过滤字段：体检、色觉、近视、外语、性别、单科成绩、特殊资格

### 下沉冲击特征

```text
sink_shock = high_rank_unadmitted_delta * major_attraction_score
```

`major_attraction_score` 由以下因素组合：

- 公办
- 低学费
- 3+2
- 本科院校专科
- 热门城市
- 就业导向专业
- 历史最低位次更靠前

### 产出

- `plan_series_features_2026.csv`
- `training_samples_2023_2025.csv`

### 验收标准

- 每个今年计划都能生成特征。
- 缺失特征有默认值和缺失标记。
- 重要标签能人工抽样校验。

## Step 6：建立基础历史趋势模型

### 要做什么

先做一个可解释、可回测的基线模型，不直接上复杂模型。

建议历史权重：

```text
base_pressure = 0.55 * pressure_last_year
              + 0.30 * pressure_two_years_ago
              + 0.15 * pressure_three_years_ago
```

注意：加权对象应优先使用“有效竞争压力坐标”，不是直接加权原始位次。

再根据计划变化、波动、小计划做修正：

```text
adjusted_pressure = base_pressure
                  + plan_change_effect
                  + sink_shock_effect
                  + volatility_penalty
```

### 产出

- `baseline_specialized_rank_model`
- 每个专业的 `p50` 预测位次。

### 验收标准

- 能完成 2025 年回测。
- 与只用去年最低位次相比，平均误差不能更差。
- 对扩招、缩招、小计划的方向判断符合常识。

## Step 7：训练非线性区间模型

### 要做什么

使用 CatBoost 或 LightGBM 做分位数回归。

推荐优先 CatBoost：

- 类别特征多；
- 样本量可能不是特别大；
- 对缺失值和类别变量处理更顺。

训练三个分位数：

- `q10`：偏难情形，即最低投档位次更靠前。
- `q50`：中性预测。
- `q90`：偏易情形，即最低投档位次更靠后。

目标变量建议使用压力坐标或等价位次坐标，再反解为当年原始位次。

### 产出

- `specialized_cutoff_q10_model`
- `specialized_cutoff_q50_model`
- `specialized_cutoff_q90_model`
- `model_feature_importance.csv`

### 验收标准

- q10 <= q50 <= q90 在“位次越小越难”的语义下保持一致。
- 80% 区间覆盖率接近 80%。
- 对新增专业、小计划专业输出更宽区间。

## Step 8：概率校准

### 要做什么

录取概率定义为：

```text
P(实际最低投档位次 >= 考生位次)
```

因为山东位次数字越小越靠前，所以如果考生位次小于或等于实际最低投档位次，则可以投档。

概率不能用简单线性差值硬算，应基于回测校准：

1. 对历史年份做滚动预测。
2. 收集模型预测概率和真实是否录取。
3. 分箱校准，例如 50%-60% 桶真实命中率应接近 55%。
4. 使用 isotonic regression 或 Platt scaling 做概率校准。

### 产出

- `probability_calibrator`
- `calibration_report.csv`
- 概率展示规则：按 5% 粒度展示，不显示 73.26% 这类伪精确数字。

### 验收标准

- 70% 概率桶真实命中率接近 70%。
- 小计划、高波动专业即使概率高，也输出风险提示。

## Step 9：生成冲稳保标签

### 标签规则

| 概率 | 标签 |
| --- | --- |
| < 20% | 不建议 |
| 20%-50% | 冲 |
| 50%-80% | 稳 |
| >= 80% | 保 |

### 附加规则

- 计划数小于 5：标记高波动。
- 新增专业：标记历史不足。
- 曾断档或进入第 3 次志愿：标记不稳定。
- 3+2 或热门公办：即使概率较高，也提示高分下沉风险。

### 产出

- 每个专业的 `label`、`volatility_label`、`risk_tips`。

### 验收标准

- 标签与概率一致。
- 风险提示能解释关键原因。
- 不给出过度确定的结论。

## Step 10：96 志愿顺序模拟

### 要做什么

山东是位次优先、遵循志愿。单项概率不能直接相乘，必须模拟志愿顺序。

推荐 Monte Carlo：

1. 对每个专业生成一次可能的今年最低投档位次。
2. 按用户 96 个志愿顺序逐一检查。
3. 第一个满足 `candidate_rank <= simulated_cutoff_rank` 的专业成为本次模拟去向。
4. 重复 5000-10000 次。
5. 统计每个专业最终去向概率和滑档概率。

### 产出

- `final_destination_probability`
- `slip_probability`
- `inversion_warnings`
- `safety_coverage`

### 验收标准

- 后序志愿不能影响前序志愿的最终去向概率。
- 如果所有保底志愿都放在前面，后面冲刺志愿的最终去向概率应接近 0，并提示倒挂。
- 滑档概率能随着保底数量增加而降低。

## Step 11：滚动回测

### 标准回测

当前已有 2022-2025 数据，建议先做两轮回测：

- 用 2022-2023 预测 2024。
- 用 2022-2024 预测 2025。
- 验证后预测 2026。

如果后续补齐 2021：

- 用 2021-2023 预测 2024。
- 用 2022-2024 预测 2025。
- 对小计划、3+2、公办专科、新增专业分别评估误差。

### 指标

| 指标 | 含义 |
| --- | --- |
| MAE | 预测最低位次平均绝对误差 |
| MdAE | 中位绝对误差，减少极端断档影响 |
| P80 coverage | 80% 区间真实覆盖率 |
| Calibration error | 概率校准误差 |
| Recall@screen | 初筛是否覆盖真实可报专业 |
| Label confusion | 冲稳保分类混淆矩阵 |
| Small-plan error | 小计划专业单独误差 |

### 验收标准

- 基线模型必须优于“直接照搬去年位次”。
- 区间模型必须比单点模型更诚实地覆盖波动。
- 概率校准报告必须能解释每个概率桶的真实命中率。

## 8. 分阶段实施计划

### Phase 1：数据盘点与口径统一

目标：确认现有数据能不能支撑最小可用模型。

任务：

1. 导入专科三表合一，生成 2026 专科计划候选表和 2025/2024/2023 历史特征。
2. 导入 2022-2025 专业录取分数和招生计划，补齐 2022 并与三表合一交叉校验。
3. 导入 2022-2026 一分一段，统一分数、累计人数、位次区间。
4. 检查院校基础信息、专业基础信息、软科/GDI、满意度、就业资料的可匹配率。
5. 建立统一 batch、education_level、plan_type、source_confidence 标签。
6. 将 EOL/掌上高考爬取结果作为校验源，不再作为 MVP 阻塞项。

产出：

- 数据质量报告。
- 训练样本候选表。
- 缺失字段清单。
- 2026 计划一致性校验报告。

### Phase 2：历史基线模型

目标：先不用复杂机器学习，跑通从历史位次到今年预测位次的闭环。

任务：

1. 构造 `plan_series_id`。
2. 为 2025 做 baseline 回测。
3. 实现历史加权 + 计划变化修正。
4. 输出单专业预测位次和冲稳保标签。

产出：

- baseline 回测报告。
- 可解释预测样例。

### Phase 3：专科等价总位次

目标：建立今年位次到去年专科环境的映射。

任务：

1. 建立 `rank_segment_flow`。
2. 估计 2025 有效专科竞争曲线。
3. 建立 2026 预估版有效竞争曲线。
4. 第 2 次志愿计划公布后更新 `K_26`。
5. 7 月 22 日本科第 1 次录取结果后发布更新版。

产出：

- 2026 -> 2025 等价位次查询表。
- 预估版/更新版对比报告。

### Phase 4：专业级区间模型

目标：给每个专业输出最低位次区间，而不只是一个点。

任务：

1. 构建专业级特征表。
2. 训练 q10/q50/q90 分位数模型。
3. 与 baseline 做对比。
4. 解释模型特征重要性。
5. 对小计划、新增、断档样本单独评估。

产出：

- 区间预测模型。
- 特征重要性报告。
- 高波动专业名单。

### Phase 5：概率校准与志愿模拟

目标：把“预测位次”转成“可用概率”和“96 志愿最终去向”。

任务：

1. 用历史回测结果训练概率校准器。
2. 将概率按 5% 粒度展示。
3. 实现 5000-10000 次志愿顺序模拟。
4. 输出滑档概率和倒挂提示。
5. 接入现有推荐/志愿表模块。

产出：

- calibrated probability。
- 志愿表模拟结果。
- 滑档风险报告。

## 10. 第一版 MVP 可以先做什么

在缺少完整本科录取后考生流动数据时，MVP 不应承诺高精度概率。建议先做：

1. 使用 2022-2025 专业级录取分数和招生计划。
2. 使用 2022-2026 一分一段表。
3. 使用专科三表合一作为 2026 专科计划主数据源。
4. 使用 `山东-2026-招生计划.xlsx` 和 EOL/掌上高考爬取结果做计划一致性校验。
5. 先用历史分段落榜率估算等价位次。
6. 使用 baseline 模型预测专业最低位次。
7. 概率先输出为粗粒度：低/中低/中/中高/高。
8. 明确标注“预估版，待本科第 1 次录取结果后更新”。

MVP 不做：

- 不展示两位小数概率。
- 不声称能准确预测新增专业。
- 不把第 3 次志愿和第 2 次志愿混用。
- 不让大模型直接猜最低位次。

## 11. 关键风险与处理

| 风险 | 影响 | 处理 |
| --- | --- | --- |
| 本科录取后剩余考生不可直接获得 | 等价位次误差大 | 用历史反推 + 分段参与率 + 7 月 22 日后更新 |
| 专业改名/拆分/合并 | 历史样本错配 | plan_series 匹配置信度 + 人工复核 |
| 小计划专业波动极大 | 概率虚高 | 扩大预测区间，增加高波动提示 |
| 新增专业无历史 | 无法训练 | 借同校同类专业、同层次学校相似专业估计 |
| 热门 3+2 和公办专科高分下沉 | 低估竞争 | 加入下沉冲击特征 |
| 数据来源混杂 | 口径不一致 | 以山东考试院官方计划为准，第三方只作补充校验 |

## 13. 成功标准

第一版系统可以认为成功，需要满足：

1. 能对 2025 做完整回测。
2. 预测最低位次误差优于“直接使用去年位次”。
3. 80% 预测区间覆盖率接近 80%。
4. 概率分桶基本校准。
5. 用户输入 2026 位次和选科后，可以输出：
   - 2025 等价总位次；
   - 可报专业初筛；
   - 每个专业预测位次区间；
   - 冲稳保标签；
   - 风险原因；
   - 96 志愿最终去向概率和滑档概率。

最重要的判断标准不是模型名字，而是：回测是否可信、概率是否校准、风险提示是否诚实。
